Using let and const

the const keyword defines a variable that cannot be reassigned
the let keyword defines a variable with restricted scope

The general rules for constructing names for variables (unique identifiers)

  • Names can contain letters, digits, underscores, and dollar signs
  • Names must begin with a letter
  • Names can also begin with $ and _
  • Names are case sensitive
  • Reserved words (like JavaScript keywords) cannot be used as names

Assignment vs Equal to

the equal sign (=) is an "assignment" operator
The "equal to" operator is written like ==

Data Types

JavaScript variables can hold numbers and strings

Declaring (Creating) Variables

to declare a variable is to assign some memory to it
by default var assigns the value undefined to any variable that is declared.
like: 

            
                var carName;    // this is now holding a value of undefined

Defining Variables

to define a variable is to assign some memory to it and pass initial data to it
like: 

            
                var carName = "Porsche";    // this is a definition = declaration + assignment

It's a good programming practice to declare all variables at the beginning of a script
Re-Declaring JavaScript Variables will not override the value

Declaring (or defining) multiple variables

You can declare many variables in one statement.
Start the statement with var and separate the variables by comma
A declaration can span on multiple lines

If you put a number in quotes, the rest of the numbers will be treated as strings, and concatenated

Since JavaScript treats a dollar sign as a letter, identifiers containing $ are valid variable names

Since JavaScript treats underscore as a letter, identifiers containing _ are valid variable names
Using the underscore is not very common in JavaScript, but a convention among professional programmers is to use it as an alias for "private (hidden)" variables
